# FairDM Plugin Cookiecutter - Quick Start Guide

## Installation

```bash
pip install cookiecutter
```

## Generate a Plugin

```bash
cookiecutter gh:FAIR-DM/fairdm-plugin-cookiecutter
```

## After Generation

```bash
cd data-visualizer

# Install dependencies
poetry install

# Open in VSCode with recommended settings
code data-visualizer.code-workspace

# Run tests
poetry run pytest
```

## Install in FairDM Project

In your FairDM project's `pyproject.toml`:

```toml
[tool.poetry.dependencies]
data-visualizer = {git = "https://github.com/johnsmith/data-visualizer"}
```

In `config/settings.py`:

```python
import fairdm

fairdm.setup(addons=["data_visualizer"])
```

## Key Points

### Template Naming

Your main template must be named to match your plugin slug:
```
templates/data_visualizer/data_visualizer.html
```

This allows FairDM to automatically discover and load it.

### URL Auto-Registration

Plugin URLs are automatically generated by FairDM. No need to create `urls.py`:
```
/dataset/123/plugins/data-visualizer/
```

### Use Existing Components

For UI consistency, use existing FairDM and Bootstrap 5 components:

```django
{% extends "fairdm/plugin.html" %}

{% block plugin %}
<div class="card">
  <div class="card-body">
    {# Your content here #}
  </div>
</div>
{% endblock %}
```

## Advanced Features

### Custom Template Tags

If you need custom template tags (advanced use case), create them in a `templatetags/` directory manually:

```python
# data_visualizer/templatetags/data_visualizer.py
from django import template

register = template.Library()

@register.simple_tag
def my_custom_tag(value):
    return value
```

### Custom URLs

If you need custom URL patterns beyond the auto-generated plugin URL:

```python
# data_visualizer/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('custom/', views.MyCustomView.as_view(), name='custom'),
]
```

Then include in your main project's `urls.py`.
